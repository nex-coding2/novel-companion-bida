## 1. 什么是 VS Code？怎么安装？

VS Code（全称 Visual Studio Code）是一个免费的代码编辑器，但它也能用来写小说——因为它的文件管理、搜索替换、多标签编辑都非常强大。

**安装步骤：**

1. 打开浏览器，访问 https://code.visualstudio.com/
2. 点击页面上的大按钮「Download」（下载）
3. 选择你的操作系统（Windows / Mac / Linux）
4. 下载后双击安装包，一路「下一步」完成安装
5. 安装完成后，打开 VS Code

> 你现在看到的 VS Code 界面分几个区域：
> - **最左边一竖条**叫「活动栏」，放各种功能图标
> - **活动栏右边**是「侧边栏」，显示文件列表等
> - **中间最大区域**是「编辑区」，你在这里写东西
> - **最下方一条**是「状态栏」，显示信息
> - **最下方一条上面**可以展开「面板区」，显示终端、问题等

## 2. 安装笔搭插件

**方法一：从 VSIX 文件安装（如果你有 .vsix 文件）**

1. 打开 VS Code
2. 按 `Ctrl+Shift+P`（Mac 按 `Cmd+Shift+P`），会弹出一个命令搜索框
3. 输入 `install from vsix`，选择「Extensions: Install from VSIX...」
4. 在弹出的文件选择窗口中，找到你的 `.vsix` 文件，点击「安装」
5. 安装完成后，左侧活动栏会出现一个书本图标，鼠标放上去显示「笔搭」

**方法二：从开发者模式安装（F5 调试）**

如果你是在开发模式下运行：
1. 用 VS Code 打开笔搭插件的项目文件夹
2. 按 `F5` 键
3. 会弹出一个新的 VS Code 窗口（标题栏写着「Extension Development Host」）
4. 这个新窗口里就加载了笔搭插件

**确认安装成功：**

左侧活动栏（最左边的竖条）应该出现一个书本形状的图标。鼠标放上去显示「笔搭」。点击它，侧边栏会显示「小说信息」「章节大纲」「正文」「人物」「设定」「伏笔」「时间线」等分块。

## 3. 安装 Kilo Code（AI 插件）

笔搭本身不包含 AI 功能，它需要配合 Kilo Code 等 AI 插件来工作。

**安装 Kilo Code：**

1. 在 VS Code 中，点击左侧活动栏的「扩展」图标（四个方块的图标），或按 `Ctrl+Shift+X`
2. 在搜索框输入 `Kilo Code`
3. 找到 Kilo Code 扩展，点击「Install」（安装）
4. 安装完成后，按提示配置你的 AI 模型 API Key（如 OpenAI / Claude / DeepSeek 等）

> 如果你用的是其他 AI 插件（如 Cline、Roo Code、Continue），也可以。笔搭通过文件与它们通信，不绑定特定插件。

---

## 4. 安装 AI 技能文件

笔搭自带 3 个技能文件，安装到 Kilo Code 后能让 AI 更好地理解小说任务。

**安装步骤：**

1. 按 `Ctrl+Shift+P`（Mac 按 `Cmd+Shift+P`），打开命令搜索框
2. 输入 `安装 AI 技能文件`，选择「笔搭: 安装 AI 技能文件」
3. 弹出一个输入框，输入 Kilo Code 的技能目录路径
   - 如果不知道路径，输入 `~/.kilo/skills` 试试
   - `~` 代表你的用户主目录（Windows 上类似 `C:\Users\你的用户名`）
4. 按回车，插件会自动复制技能文件
5. 看到提示「已复制 3 个技能文件」后，重启 Kilo Code（关掉再重新打开 VS Code 窗口）

**3 个技能文件的作用：**

| 技能 | 作用 |
|------|------|
| 小说审稿 | 让 AI 知道怎么读取笔搭的任务文件、怎么生成审稿建议 |
| 小说灵感辅助 | 让 AI 提供情节方案而不是代写正文 |
| 设定一致性检查 | 让 AI 检查人物、设定、战力是否前后矛盾 |

## 5. 新建你的第一个小说项目

**步骤：**

1. 先用 VS Code 打开一个空文件夹（这个文件夹将存放你整本小说的所有文件）
   - 菜单栏 → 「文件」→「打开文件夹」→ 选择或新建一个空文件夹
2. 按 `Ctrl+Shift+P`，输入 `新建小说项目`，选择「笔搭: 新建小说项目」
3. 按提示逐步输入：

   | 步骤 | 你要输入什么 | 示例 |
   |------|-------------|------|
   | 小说名称 | 你的书名 | 我的修仙路 |
   | 作者 | 你的笔名（可留空） | 张三 |
   | 题材 | 从列表中选择 | 玄幻 |
   | 篇幅 | 从列表中选择 | 网文长篇 |
   | 目标字数 | 全书目标字数 | 800000 |
   | 每日目标字数 | 每天计划写多少字 | 3000 |

4. 完成后，VS Code 会在当前文件夹下自动创建完整的项目目录结构

**创建完成后你会看到什么：**

左侧的「资源管理器」（文件树）里会出现一堆文件夹和文件：

```
你的小说文件夹/
├── novel.yaml              ← 小说配置（书名、作者等）
├── bible/                  ← 故事圣经
│   ├── premise.md          ← 故事前提（含故事简介）
│   └── style-guide.md      ← 风格指南
├── characters/             ← 人物卡（现在为空）
├── world/                  ← 设定卡（现在为空）
├── outline/                ← 大纲
│   └── main-outline.md     ← 总大纲
├── chapters/               ← 章节正文（现在为空）
├── foreshadowing/          ← 伏笔
│   ├── unresolved/         ← 待回收
│   └── resolved/           ← 已回收
├── timeline/               ← 时间线（现在为空）
├── .novel-ai/              ← 笔搭的工作目录（AI 任务等）
│   ├── tasks/              ← AI 任务文件
│   ├── results/            ← AI 审稿结果
│   ├── schemas/            ← 格式校验文件
│   └── config.yaml         ← 笔搭配置
└── .vscode/
    └── settings.json       ← 工作区设置
```

同时，左侧「笔搭」侧边栏的「小说信息」分块会显示你刚填的书名、作者、题材等信息。

> 不同题材会生成不同的模板。例如选「玄幻」会包含力量体系模板，选「悬疑」会包含线索设计模板。

## 6. 认识界面：侧边栏 7 个分块

点击左侧活动栏的「笔搭」图标（书本形状），侧边栏会显示 7 个分块。从上到下：

### ① 小说信息
显示书名、作者、题材、卷数、章节数、总字数、故事梗概等。右键信息项可编辑。

### ② 章节大纲
列出所有章节，按卷分组。**点击某个章节 → 打开该章的大纲文件**（outline.md），里面是你填写本章计划的地方。

### ③ 正文
列出所有章节，按卷分组。**点击某个章节 → 打开该章的正文文件**（draft.md），这里是你写正文的地方。

> 章节大纲和正文是两个独立的分块，但列出的是同一批章节。区别在于点击时打开的文件不同：
> - 点「章节大纲」里的章节 → 打开大纲（规划本章写什么）
> - 点「正文」里的章节 → 打开正文（实际写内容）

### ④ 人物
列出所有人物卡，按角色类型分组（主角/重要角色/反派/配角等）。点击人物 → 打开该人物的人物卡文件。

### ⑤ 设定
列出所有设定卡，按分类分组（规则/力量体系/地点/组织/物品等）。点击设定 → 打开该设定的文件。

### ⑥ 伏笔
列出所有伏笔，按状态分组（待回收/已回收）。点击伏笔 → 打开该伏笔的文件。右键未回收伏笔 → 可选择「回收伏笔」。

### ⑦ 时间线
列出所有时间线。点击时间线 → 打开该时间线文件（含 Mermaid 图）。

### 如何调整各分块大小？

把鼠标放到两个分块之间的分界线上，鼠标会变成上下双向箭头 ⬍，此时按住鼠标上下拖动即可调整高度。

### 如何隐藏不需要的分块？

右键点击任意分块的标题，选择「隐藏」即可隐藏该分块。要恢复，右键活动栏的笔搭图标 → 选择「重置视图」或到设置中重新开启。

## 7. 认识界面：底部创作工作流面板

VS Code 底部有一个可展开的面板区（如果看不到，按 `Ctrl+J` 切换显示）。

在底部面板中，你会看到「创作工作流」标签页。点开后显示一个横向流程图：

```
📝计划  →  ✏️写作  →  🔍检查  →  🔧修订  →  ✅归档
```

- **当前所在阶段**会高亮显示（蓝底白字）
- **已完成的阶段**显示打勾 ✓（绿底）
- **未到达的阶段**灰色显示
- 下方有操作按钮，根据当前阶段不同：
  - 计划阶段：「填写章节计划」→ 打开大纲
  - 写作阶段：「生成 AI 审稿任务」→ 触发 AI 审稿
  - 归档后：「新建下一章」
- 切换编辑器（比如从第 1 章切到第 2 章）时，流程图会自动更新到对应章节的进度

> 如果底部面板看不到「创作工作流」标签，点击底部面板右上角的「...」或下拉菜单，勾选「创作工作流」。

## 8. 认识界面：底部状态栏

VS Code 最底部一条横线是状态栏。笔搭会在左侧显示：

```
📖 ch012 | 3245字 | 全书15.3万 | 今日+1280 | ██████░░ 64%
```

各部分含义：
- `ch012` — 当前编辑的章节 ID
- `3245字` — 当前章节字数
- `全书15.3万` — 全书总字数
- `今日+1280` — 今天新增字数
- `██████░░ 64%` — 今日写作目标进度条（写满 64% 的每日目标）

写正文时，字数会实时更新。写作满一定时间后，状态栏会变成审稿提醒：

```
⏰ 已写 30 分钟 / +1280 字 · 该审稿了
```

点击它即可生成 AI 审稿任务。

## 9. 创建章节并写正文

### 创建章节

1. 在侧边栏「章节大纲」分块，点击标题右边的「+」按钮
   - 或按 `Ctrl+Shift+P`，输入 `新建章节`
2. 输入章节标题，如「第一章 初入江湖」
3. 输入所属卷（可留空，留空归入「默认卷」）
4. 章节自动创建，同时打开正文文件

### 填写章节大纲

创建章节后，侧边栏「章节大纲」里会出现这个章节。点击它打开 `outline.md`，你会看到模板：

```markdown
# 第一章 初入江湖

## 本章目标


## 出场人物


## 地点


## 时间


## 冲突点


## 高潮情节


## 结尾钩子


## 场景设计


## 备注

```

在各个标题下面填写内容。例如：

```markdown
## 本章目标

主角下山，初入江湖，遇到第一个冲突。

## 出场人物

- 林澈（主角）
- 老乞丐（引路人）

## 地点

- 清风山脚下
- 山下小镇
```

### 写正文

在侧边栏「正文」分块，点击对应章节，打开 `draft.md`。文件开头有几行 `---` 包裹的配置信息（不要删），下面空着的地方就是你写正文的地方：

```markdown
---
章节: ch001
标题: 第一章 初入江湖
状态: drafting
---

清风山的晨雾还没散尽，林澈已经背好行囊站在山门前。
```

直接在 `---` 下面的空白处写你的小说正文。每写一个字，底部状态栏的字数都会实时更新。

> 写正文时，编辑器可能会在某些词下面出现波浪线（红色、黄色、蓝色）。这是笔搭的本地检查功能：
> - **黄色波浪线** — 可能是错别字或标点问题
> - **蓝色波浪线** — 可能是重复词或长句
> - 鼠标放到波浪线上会显示具体问题

## 10. 什么是 Markdown？怎么编辑 .md 文件？

### 什么是 Markdown？

Markdown 是一种简单的文本格式。你写的 `.md` 文件本质就是纯文本，但用一些简单符号表示格式：

| 你输入的 | 显示效果 | 说明 |
|----------|---------|------|
| `# 标题` | 大标题 | 一个 `#` 是一级标题 |
| `## 小标题` | 小标题 | 两个 `##` 是二级标题 |
| `**粗体**` | **粗体** | 两边各两个星号 |
| `- 列表项` | • 列表项 | 减号开头 |
| `1. 有序列表` | 1. 有序列表 | 数字点开头 |
| `> 引用` | 缩进引用 | 大于号开头 |

### 怎么编辑 .md 文件？

直接像写普通文本一样写就行。Markdown 符号是可选的，不写也完全没问题。你可以：

- 直接打字写正文
- 用 `##` 来分段
- 用 `-` 来列清单
- 也可以完全不用任何符号，纯文本写

> 笔搭生成的模板里的 `## 本章目标`、`## 出场人物` 这些只是建议结构，你可以随意修改、增删。

### Front Matter（文件开头的配置区）

你会看到 `.md` 文件开头有这样的内容：

```
---
章节: ch001
标题: 第一章 初入江湖
状态: drafting
---
```

这叫 Front Matter，是文件的配置信息。**不要删除或修改这个区域**（除非你知道在做什么），笔搭靠它来识别文件。

## 11. 预览 Markdown（含时间线 Mermaid 图）

### 为什么要预览？

你在编辑器里看到的是 Markdown 源码（带 `#`、`-` 等符号）。预览模式会把这些符号渲染成实际格式，让你看到最终效果。

### 怎么预览？

1. 打开任意 `.md` 文件
2. 按 `Ctrl+Shift+V`（Mac 按 `Cmd+Shift+V`）
3. 编辑器会切换到预览模式，显示渲染后的效果
4. 再按一次 `Ctrl+Shift+V` 或关掉预览标签，回到编辑模式

### 并排预览（边写边看）

1. 打开 `.md` 文件
2. 点击编辑器右上角的「打开侧边预览」按钮（一个带放大镜的图标）
   - 或按 `Ctrl+K` 然后按 `V`（先按 Ctrl+K 松开，再按 V）
3. 左边显示源码，右边显示预览，实时同步

### 时间线 Mermaid 图怎么看？

时间线文件里有一段用三个反引号包裹的 Mermaid 代码：

````markdown
```mermaid
graph LR
  N1["初入江湖\n永宁三年三月初五"]
  N2["发现族徽\n永宁三年三月初七"]
  N1 --> N2
```
````

在编辑模式下，你看到的就是上面这段代码。**切换到预览模式后**，这段代码会被渲染成一个有向箭头图：

```
[初入江湖] ──→ [发现族徽]
永宁三年三月初五    永宁三年三月初七
```

> 如果预览模式看不到 Mermaid 图（只看到代码），说明你的 VS Code 版本较旧或缺少 Mermaid 支持。解决方法：
> - 升级 VS Code 到 1.84 或更高版本
> - 或安装扩展「Markdown Preview Mermaid Support」（在扩展商店搜索安装）

## 12. 管理人物卡

### 新建人物卡

1. 在侧边栏「人物」分块，点击标题右边的「+」按钮
2. 输入人物姓名，如「林澈」
3. 选择角色类型（主角/重要角色/反派/配角等）
4. 人物卡自动创建并打开

### 人物卡文件内容

```markdown
---
编号: lin-che
姓名: 林澈
角色类型: protagonist
状态: alive
创建时间: 2026-07-12T18:20:00Z
更新时间: 2026-07-12T18:20:00Z
---

# 林澈

## 基本信息

- 别名：
- 年龄：
- 性别：
- 阵营：

## 外貌


## 性格


## 背景


## 能力


## 人物关系

```

在各标题下填写人物信息。这些信息会被笔搭打包给 AI，帮助 AI 审稿时判断人物行为是否一致。

### 编辑人物卡

随时在侧边栏「人物」分块点击人物名，打开文件直接编辑。保存后笔搭自动更新索引。

## 13. 管理设定卡

### 新建设定卡

1. 在侧边栏「设定」分块，点击标题右边的「+」按钮
2. 输入设定名称，如「玄火令」
3. 选择分类（规则/力量体系/地点/组织/物品/历史/其他）
4. 选择重要程度（关键/高/中/低，可跳过）
5. 设定卡自动创建并打开

### 设定卡文件内容

```markdown
---
编号: xuanfire-token
名称: 玄火令
分类: item
重要程度: high
创建时间: 2026-07-12T18:20:00Z
更新时间: 2026-07-12T18:20:00Z

## 14. 管理伏笔

### 新建伏笔

1. 在侧边栏「伏笔」分块，点击标题右边的「+」按钮
2. 输入伏笔标题，如「铜铃声」
3. 输入伏笔描述（选填），如「林澈每次进入旧宅都听到铜铃声」
4. 输入埋设章节（选填），如「ch008」
5. 选择重要程度（高/中/低）
6. 伏笔文件自动创建在 `foreshadowing/unresolved/` 目录下

### 伏笔文件内容

```markdown
---
编号: tong-ling-sheng
名称: 铜铃声
状态: pending
重要程度: high
埋设章节: ch008
回收章节:
创建时间: 2026-07-12T18:20:00Z
更新时间: 2026-07-12T18:20:00Z
---

# 铜铃声

## 描述

林澈每次进入旧宅都听到铜铃声

## 相关人物

（待补充）

## 预期回收

（待补充）

## 回收情况

（未回收）
```

### 查看伏笔

侧边栏「伏笔」分块会按状态分组显示：
- **待回收 (N)** — 展开后显示所有未回收伏笔
- **已回收 (M)** — 展开后显示所有已回收伏笔

点击伏笔名 → 打开伏笔文件。

### 回收伏笔

当你写到回收某个伏笔的章节时：

1. 在侧边栏「伏笔」分块，右键点击待回收的伏笔
2. 选择「回收伏笔」
3. 输入回收章节（如 `ch020`）
4. 伏笔状态自动改为「已回收」，文件从 `unresolved/` 移到 `resolved/` 目录
5. 树视图自动刷新，该伏笔从「待回收」移到「已回收」分组

## 15. 管理时间线

### 新建时间线

1. 在侧边栏「时间线」分块，点击标题右边的「+」按钮
2. 输入时间线名称，如「第一卷主线」或「ch012时间线」
3. 输入节点数量（你想要几个时间节点），如「3」
4. 逐个输入每个节点的信息：
   - 节点名称（如「初入江湖」）
   - 故事时间（如「永宁三年三月初五」，选填）
   - 事件（如「林澈下山」，选填）

## 16. 编辑小说信息和故事梗概

### 编辑小说信息

书名、作者、题材等创建后可以修改：

1. 在侧边栏「小说信息」分块，右键点击任意信息项
2. 选择「编辑小说信息」
   - 或点击底部的「编辑小说信息」按钮
3. 按向导逐步修改：书名 → 作者 → 题材 → 篇幅 → 风格 → 目标字数 → 每日目标
4. 修改完成后自动保存到 `novel.yaml`，侧边栏实时更新

### 编辑故事梗概

故事梗概是你小说的一句话简介（类似封面上的推荐语）：

1. 在侧边栏「小说信息」分块，点击底部的「编辑故事梗概」按钮
   - 或按 `Ctrl+Shift+P`，输入 `编辑故事梗概`
2. 弹出输入框，输入你的故事梗概
   - 例如：「一个被废的天才重生回少年时代，凭借前世记忆碾压一切」
3. 按回车保存，梗概写入 `bible/premise.md` 的「故事简介」段落
4. 侧边栏「小说信息」分块的「故事梗概」行会显示截取后的预览

## 17. AI 审稿完整流程（与 Kilo Code 配合）

这是笔搭最核心的功能。完整流程如下：

### 第 1 步：写正文

在 `draft.md` 中写正文。写到一定时间（默认 30 分钟）或一定字数（默认 1000 字）后，状态栏会提示：

```
⏰ 已写 30 分钟 / +1280 字 · 该审稿了
```

### 第 2 步：生成 AI 审稿任务

1. 点击状态栏的提醒，或按 `Ctrl+Shift+P` 输入 `生成 AI 审稿任务`
2. 选择任务模板（通常选 `full-review` 全章审稿）
3. 选择权限级别（通常选「建议修订」）
4. 笔搭自动：
   - 打包当前章节的正文、大纲、相关人物卡、设定卡为上下文
   - 生成 3 个文件到 `.novel-ai/tasks/任务ID/` 目录：
     - `task.json` — 任务元数据
     - `instruction.md` — 给 AI 的指令
     - `context.md` — 上下文内容

### 第 3 步：交给 Kilo Code

1. 任务生成后，弹出通知：「AI 审稿任务已生成：task-XXXXXXXX-NNN」
2. 点击「复制路径给 Kilo Code」按钮
3. 路径已复制到剪贴板
4. 打开 Kilo Code 的对话框（通常在侧边栏或底部）
5. 输入：
   ```
   请阅读 <粘贴的路径>/ 下的任务文件并执行审稿
   ```
   例如：
   ```
   请阅读 D:/我的小说/.novel-ai/tasks/task-20260712-001/ 下的任务文件并执行审稿
   ```
6. Kilo Code 会读取任务文件，执行审稿分析

### 第 4 步：Kilo Code 生成结果

Kilo Code 审稿完成后，会在 `.novel-ai/results/任务ID/` 目录下生成：
- `review.md` — 人类可读的审稿报告
- `suggestions.json` — 结构化修改建议

### 第 5 步：自动导入

笔搭会自动检测到结果文件并导入。你会看到通知：
```
AI 审稿结果已导入：3 条建议待审阅
```

点击「打开审阅面板」即可查看建议。

> 如果自动导入没有触发，可以手动导入：`Ctrl+Shift+P` → `导入 AI 审稿结果` → 选择 `suggestions.json` 文件

## 18. 审阅 AI 建议（接受/拒绝/合并）

### 打开审阅面板

1. 点击通知上的「打开审阅面板」
   - 或按 `Ctrl+Shift+P` 输入 `打开审阅面板`
   - 或点击状态栏的「🔍 待审阅 N」
2. 审阅面板在编辑区右侧打开

### 建议卡片

每条建议显示为一张卡片：

```
┌──────────────────────────────────────────────┐
│ 建议 #1  [战力/能力] [严重]                    │
│──────────────────────────────────────────────│
│ 原文：裴玄拔剑挡下三名黑衣人的围攻。           │
│ 建议：裴玄以左手短刃牵制黑衣人。               │
│ 原因：第11章裴玄右肩受伤，惯用右手剑。          │
│──────────────────────────────────────────────│
│ [✓ 接受]  [✗ 拒绝]  [✎ 合并编辑]              │
└──────────────────────────────────────────────┘
```

### 三种操作

| 操作 | 效果 | 什么时候用 |
|------|------|-----------|
| **接受** | 用 AI 建议的文本替换原文 | AI 建议完全正确，直接采纳 |
| **拒绝** | 忽略该建议，不修改 | AI 建议不对，不需要 |
| **合并编辑** | 打开对比视图，手动编辑 | AI 建议部分正确，需要修改后再采纳 |

### 合并编辑怎么用

1. 点击「合并编辑」
2. VS Code 打开一个左右分屏的对比视图：
   - 左边：原文
   - 右边：AI 建议（可编辑）
3. 在右边直接修改 AI 建议的文本
4. 按 `Ctrl+S` 保存
5. 你编辑后的文本会替换原文

### 批量操作

- 「接受所有可自动应用」：一键接受所有标记为可自动应用的建议（通常是错别字、标点等小问题）
- 「拒绝所有建议」：一键拒绝所有待处理建议

### 处理进度

面板顶部显示进度：`已处理 3/8 | 待处理 5`

全部处理完后显示：`全部建议已处理 ✓`

## 19. 章节归档与下一章

### 归档当前章节

当一章写完、审稿完毕、修改满意后：

1. 在侧边栏「章节大纲」或「正文」分块，右键点击该章节
2. 选择「归档章节」
3. 确认后，章节状态变为「已归档」(✅)
4. 底部工作流面板显示全部完成

### 开始下一章

1. 归档后，底部工作流面板显示「新建下一章」按钮
2. 点击它，或在侧边栏「章节大纲」点「+」按钮
3. 输入新章节标题
4. 新章节从「计划」阶段开始，循环重新开始

### 完整循环

```
计划（填大纲）→ 写作（写正文）→ 检查（AI审稿）→ 修订（接受/拒绝建议）→ 归档 → 下一章
```

每章都经历这个循环。这就是「可持续生成式创作工作流」。

## 20. 常见问题

### Q: 侧边栏的分块看不到怎么办？

**A:** 点击左侧活动栏的笔搭图标（书本形状）。如果某些分块被隐藏了，右键活动栏的笔搭图标 → 选择显示隐藏的视图。

### Q: 侧边栏分块太多看不下怎么办？

**A:** 右键不需要的分块标题 → 选择「隐藏」。只保留你常用的。需要时再恢复。

### Q: 底部「创作工作流」面板看不到怎么办？

**A:** 按 `Ctrl+J` 切换底部面板显示。如果面板里没有「创作工作流」标签，点击面板右上角的下拉菜单（或「...」），勾选「创作工作流」。

### Q: 写正文时出现波浪线是什么？

**A:** 这是笔搭的本地检查：
- 黄色 = 错别字或标点问题
- 蓝色 = 重复词或长句
鼠标放到波浪线上看具体问题。这些不是 AI 检查，是本地规则检查，实时运行。

### Q: 生成 AI 任务后，Kilo Code 不知道怎么审稿？

**A:** 确认你已安装技能文件（见第 4 节）。安装后在 Kilo Code 中输入完整指令：
```
请阅读 <任务路径>/ 下的任务文件并执行审稿
```
确保路径正确（可以从通知的「复制路径」按钮获取）。

### Q: AI 审稿结果没有自动导入？

**A:** 检查 `.novel-ai/results/任务ID/` 目录下是否有 `suggestions.json` 文件。如果有但没自动导入，手动导入：`Ctrl+Shift+P` → `导入 AI 审稿结果` → 选择该文件。

### Q: 时间线 Mermaid 图看不到？

**A:** 打开时间线 `.md` 文件后，按 `Ctrl+Shift+V` 切换到预览模式。如果还是看不到图，升级 VS Code 到 1.84+ 或安装扩展「Markdown Preview Mermaid Support」。

### Q: 字数统计不准确？

**A:** 笔搭默认：中文字每个算 1 字，英文单词每个算 1 字，不计空格，计标点。可在设置中修改：`Ctrl+,` 搜索 `novelCompanion.stats` 调整。

### Q: 文件删除了但侧边栏还显示？

**A:** 笔搭会自动监听文件变化并刷新。如果没刷新，按 `Ctrl+Shift+P` → `检测当前项目` 手动刷新。

### Q: 怎么修改笔搭的设置？

**A:** 按 `Ctrl+,` 打开设置，搜索 `novelCompanion`，可以看到所有可配置项：审稿提醒间隔、字数统计规则、本地检查开关等。

### Q: 小说数据存在哪里？会上传到云端吗？

**A:** 所有数据存在你本地的项目文件夹中，全是 `.md`、`.yaml`、`.json` 文件。不会上传到任何云端。你可以用 Git 做版本管理，也可以直接复制文件夹备份。

## 附录：R5 进阶能力

### A.1 导出成品（EPUB / DOCX）

写完后可把全书导出为可在阅读器或 Word 打开的格式：`Ctrl+Shift+P` → `笔搭: 导出全书` → 选择 `EPUB` 或 `Word`。导出文件在 `.novel-ai/exports/` 下。EPUB 含目录与排版样式，DOCX 可在 Word/WPS 继续编辑。

### A.2 MCP Server（让 AI 直接读你的小说数据）

`Ctrl+Shift+P` → `笔搭: 安装 MCP Server`，会把一个只读 MCP 脚本写入 `.novel-ai/mcp/`，并生成 Kilo Code / Cline / Continue 的接入配置与指引。按指引把配置贴到对应 AI 客户端后，AI 就能查询你的章节、人物、伏笔、时间线等数据（**只读**，不能通过它改正文）。

### A.3 本地加密保险箱

对敏感草稿可加密保存：打开该文件 → 编辑器右键 → `笔搭: 加密入箱` → 输入并确认口令。原文会被移入回收站，加密密文存于 `.novel-ai/backups/vault/`。需要时 `笔搭: 解密还原` 选文件输入口令还原。**口令不存储，请牢记，遗忘无法恢复。**

### A.4 AI 直连校对（可选）

若不想每次都靠 AI 插件审稿，可启用 AI 直连：先 `笔搭: 设置 AI 直连 API Key` 填入 OpenAI 兼容端点的 Key，再到设置开启 `novelCompanion.aiDirect.enabled` 并配置 `apiBaseUrl` / `apiModel`。之后 `笔搭: AI 直连校对章节` 即可对当前章节一键校对，**结果仍导入审阅面板，需你确认后才落笔**（与文件协议理念一致）。API Key 存于本机 SecretStorage，不写入配置明文。
