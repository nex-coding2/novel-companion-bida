# 小说设定一致性检查技能

## 何时使用此技能

当用户要求你检查小说的设定一致性、战力体系、时间线矛盾、或跨章节一致性问题时使用。

## 检查范围

1. **人物一致性**：性格、能力、关系是否前后矛盾
2. **设定一致性**：能力体系、地理、组织、道具使用是否合规
3. **战力检查**：是否存在无铺垫击败强者的战力崩塌
4. **时间线**：事件先后是否矛盾、伤势恢复是否过快
5. **伏笔**：是否有应回收而未回收的伏笔

## 执行流程

1. 读取 `.novel-ai/tasks/<task-id>/` 下的任务文件
2. 对照 context.md 中的人物卡和设定卡进行检查
3. 在 `.novel-ai/results/<task-id>/suggestions.json` 中输出问题
4. type 使用：character, setting, power, logic
5. 权限为 review_only — replacement 可为空（仅指出问题）

## 注意事项

- 每条建议的 `evidence` 字段应列出依据文件路径
- severity 为 critical 或 high 的问题必须列出
- 如果没有发现问题，告知用户「未发现一致性矛盾」
